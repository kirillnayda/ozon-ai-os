# Правила разработки Ozon AI OS

- Отвечать и документировать решения на русском языке; имена API и кода — на английском.
- Никогда не добавлять `.env`, токены, API-ключи, дампы БД и PDF-этикетки в Git.
- Любая операция записи в Ozon запрещена при `LIVE_MODE=false`.
- При `LIVE_MODE=true` запись разрешена только после подтверждения разрешённого Telegram-чата.
- Не угадывать схемы Seller API. Новый endpoint добавляется только вместе со ссылкой/датой проверки контракта и contract-тестом.
- Все внешние строки экранировать перед отправкой в Telegram HTML.
- Не логировать заголовки авторизации, Telegram URL или содержимое секретов.
- Бизнес-логика не должна находиться в `app/main.py` или transport-классах.
- Репозитории определяются интерфейсами; SQLite — адаптер первого релиза.
- Изменения должны сопровождаться тестами. Локальная проверка: `python scripts/check.py`.
- Updater принимает только фиксированное действие обновления доверенного репозитория; произвольные команды запрещены.
- Developer Agent работает только в `feature/dev-<id>-<slug>`, не выполняет merge или deploy и не читает `.env`.
- Developer Agent запускает Codex только с `workspace-write`; dangerous bypass-флаги запрещены.
- Push Developer Agent разрешён только как точный refspec сохранённой ветки готовой задачи.
